### [프로젝트 1]

### 1-1 동일한 자동화 워크플로우 개요
본 워크플로우는 고객의 문의 유형(일반 문의 vs 견적 요청)에 따라 자동으로 데이터를 분류하여 상이한 템플릿의 이메일 알림을 보조하는 CS 인바운드 자동화 시스템입니다.

     구성 형태

     Trigger (트리거): Google Forms에 새로운 설문 응답 접수

     Action 1 (액션 1): Google Sheets 스프레드시트에 해당 고객 정보 및 문의 데이터 자동 추가 기록

     Filter / Router (분기): 문의 유형 필터링 (경로 A: 일반 문의 / 경로 B: 견적 요청)

     Action 2 (액션 2): 각각의 경로에 맞춘 개인화 및 시각적 CSS가 적용된 Gmail 메일 발송

# 1-2 / 1-3 각 도구별 워크플로우 구성/실행 화면

### Make (메이크) 워크플로우 구성
시각적 구조: 좌에서 우로 흐르는 유동적인 원형 노드(Node) 형태

트리거: Google Forms (Watch Responses) 모듈이 작동 (폴링 주기 기반)

연결 및 분기

     1. Google Sheets (Add a Row) 추가 후, 중앙의 녹색 Router 모듈을 통해 위아래 2갈래의 경로로 깔끔하게 나뉩습니다.

     2. 각 갈래의 중간 연결선에 필터(견적 요청, 일반 문의)를 직관적으로 걸어두어 조건 분기를 한눈에 알아볼 수 있습니다.

     3.라우터(Router) 모듈의 역할: Make는 데이터를 2개 이상의 경로로 쪼갤 때 Router라는 전용 도구를 사이에 배치합니다. 이 라우터는 들어온 데이터(입력값)의 복사본을 여러 갈래로 동시에 복사하여 던져주는 분배기 역할을 수행시켰습니다.

필터(Filter) 게이트 방식

Make는 라우터에서 갈라져 나가는 ‘물리적인 연결선(선 자체)’에 조건 필터를 설정합니다.

예를 들어 구글 폼에서 들어온 답변 변수가 [견적 요청]일 때만 흐르게 만들고 싶다면, 두 모듈 사이를 이어주는 실선을 마우스로 클릭하여 변수 = 견적 요청 필터를 걸어둡니다.

데이터가 선을 지나갈 때 이 필터 조건을 통과하면 다음 모듈로 진행되고, 조건이 맞지 않으면 그 즉시 해당 경로의 흐름은 통과하지 못하고 조용히 멈춥니다(Fail이 아니라 정상 필터링 종료).

Zapier (재피어) 워크플로우 구성
시각적 구조: 위에서 아래로 흐르는 수직 계층형 탑다운(Top-down) 카드 형태

트리거: 1. Google Forms (New Form Response)로 시작

연결 및 분기:

     1. Google Sheets (Change Sheet Properties)를 거친 후, 3. Split into paths (Paths 기능)를 통해 좌우 Path A와 Path B 하위 갈래로 나뉩니다.

     2. 각 하위 갈래 밑에 조건 필터링(4/6. Path conditions)과 최종 지메일 액션(5/7. Send Email)이 직렬 구조로 배치되어 있습니다.

     3.Zapier는 3단계 카드에서 Paths라는 액션 단계를 추가해 분기를 나눕니다. 이 기능을 실행하면 화면이 한 단계 옆으로(또는 세부 브랜치로) 확장되면서 Path A, Path B와 같은 가상의 개별 독립 공간을 생성겼습니다.

패스(Path) 작동 방식

각 Path(예: Path A)를 클릭하고 들어가서 "어떤 조건일 때 이 경로를 실행할 것인가?"를 설정합니다. (예: 문의유형이 견적과 정확히 일치할 때)

조건이 맞아서 해당 Path 안으로 진입하면, 그 아래에 덧붙여진 액션(예: Gmail 발송, Slack 메시지 전송 등)이 순차적으로 실행됩니다.

### [프로젝트 2]

### 2-1 자동화할 반복 업무 1개 정의

주제명: 인바운드 PC 견적 및 고객 문의 자동 분류·알림 시스템 구축

목표: 고객이 웹 폼을 통해 제출한 PC 견적 및 문의 요청 데이터를 수작업 없이 실시간으로 분류하고, 담당자 및 고객에게 맞춤형 알림을 즉시 발송하여 리드(Lead) 대응 시간을 획기적으로 단축함.

## 자동화 아키텍처 흐름

| [1단계: 트리거] | [2단계: 로깅]| [3단계: 라우팅/필터] | [4단계: 최종 액션] |
| :-: |  :-:  |  :-:  | :-: |
| 고객이 구글 폼 제출 |구글 시트에 행 추가 | 문의 유형 조건 판별 | 지메일(Gmail) 발송 |

Step 1. 데이터 수집 (Trigger):
고객이 홈페이지나 링크를 통해 Google Forms(구글 폼)으로 PC 견적 요청서 또는 문의 사항을 제출하면 자동화 플랫폼이 이 데이터를 실시간으로 감지합니다.

Step 2. 데이터베이스 로깅 (Action 1):
수집된 고객의 이름, 연락처, 이메일, 예산, 문의 내용을 Google Sheets(구글 시트)의 상담 관리 대장에 한 줄씩 자동으로 기록합니다.

Step 3. 조건부 분기 (Filter / Router):
고객이 선택한 '문의 종류' 데이터에 따라 경로를 나눕니다.

    조건 A (일반 문의): 배송, AS, 재고 문의 등 단순 CS 건

    조건 B (견적 요청): 특정 예산(예: 150만 원 상당) 및 조립 PC 부품 매칭 요청 및 배송일 설정 건

### 2-2 자동화 도구 1개 선정 및 선정 이유

     선정 도구: Make PC 견적 및 문의 처리 자동화를 위해 Zapier 대신 Make을 선정하였습니다.

1.  복잡한 다중 분기(라우팅)의 우수성

    PC 견적 요청은 고객의 예산(예: 100만 원 이하, 200만 원 이상)이나 구매 목적(사무용, 게임용, 영상편집용)에 따라 내부 담당자가 지정되거나 다른 템플릿의 이메일이 나가야 합니다.

    Make는 시각적인 Router(라우터) 모듈을 통해 이러한 다중 분기 흐름을 한 화면에서 직관적으로 드래그 앤 드롭하며 설계할 수 있어, 위계 구조로 화면을 전환해야 하는 Zapier보다 설계와 유지보수가 훨씬 간단했습니다.

2.  Zapier는 분기(Paths) 기능을 쓰려면 반드시 월 $20 이상의 유료 요금제(Professional 이상)를 결제해야 합니다. 반면, Make는 무료 플랜에서도 라우터와 다단계(Multi-step) 시나리오를 아무런 제한 없이 구성할 수 있어 초기 구축 비용이 전혀 들지 않습니다.

3.  Make는 실행 히스토리에서 숫자 말풍선(Bubble)을 통해 어떤 데이터가 입력되었고 가공되었는지 시각적으로 직접 추적할 수 있어 디버깅 속도가 압도적으로 빠릅니다.

### 2-3 워크플로우(Workflow) 설계 설명

1단계: Trigger (데이터 수집)

     사용 모듈: Google Forms - Watch Responses (실시간 응답 감지)

     설명: 고객이 웹사이트에 삽입된 구글 폼 링크를 통해 이름, 연락처, 이메일, 문의 유형(일반 문의/조립 PC 견적), 희망 예산, 사용 용도를 작성하여 제출하면 Make 시나리오가 즉시 작동합니다.


2단계: Action 1 (상담 대장 기록)
     
     사용 모듈: Google Sheets - Add a Row (행 추가)

     설명: 제출된 데이터를 상담_종합_대장.xlsx 구글 시트에 순서대로 기록합니다. 누락 없는 데이터 백업과 추후 매출 분석을 위한 기초 데이터를 자동으로 로깅하는 단계입니다.

3단계: Router (조건별 경로 분기)

     사용 모듈: Router & Filter (라우터 및 연결선 필터)

     설명: 폼에서 수집된 '문의 유형' 변수값을 판별하여 물길을 나누는 핵심 구간입니다.

     경로 A [일반 문의]: 문의 유형이 '단순 문의(재고, 배송, A/S)'인 경우에만 이 경로의 필터가 열립니다.

     경로 B [견적 요청]: 문의 유형이 '조립 PC 견적 요청'인 경우에만 이 경로의 필터가 열립니다.

4단계: 최종 Action (경로별 맞춤형 대응)

     경로 A [일반 문의] 실행 시: 사용 모듈: Gmail - Send an Email

     일반 문의 내용이 CS 관계자의 이메일로 전송되어 빠른 시간 안에 내용을 파악 후 조치가 가능합니다.
     
     경로 B [견적 요청] 실행 시: 사용 모듈 Gmail - Send an Email
     
     고객 인적 사항 및 이메일의 상세한 견적 요청 내용을 포함하여 금액에 맞는 제품을 빠르게 준비할 수 있습니다.
